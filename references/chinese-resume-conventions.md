# 中文简历排版惯例

## 字体规范

### 推荐字体

| 用途 | 中文字体 | 英文/数字/日期 |
|-----|---------|--------------|
| 正文 | 宋体 | Times New Roman |
| 标题 | 微软雅黑 或 黑体 | Times New Roman |
| 备选正文 | 等线 | Times New Roman |

**原则**：中英文混排时，中文用中文字体，英文字符和数字自动用 Times New Roman。
Word 里的做法：正文段落样式设中文字体=宋体，西文字体=Times New Roman。

### 禁用字体

- **Calibri**：Windows 默认英文字体，中文显示效果差，专业度低
- **楷体**：过于文艺，不适合求职简历
- **华文行楷 / 华文彩云 / 方正舒体**：装饰性字体，HR 看起来头疼
- **Comic Sans / Arial**：外观太随意，不适合正式简历

## 字号规范

| 区域 | 字号 | 说明 |
|-----|-----|-----|
| 姓名 | 20–24pt | 要够大，一眼定位 |
| Section 标题（如"教育背景"） | 12pt | 加粗；可加下划线分割线 |
| 正文内容 | 10.5–11pt | 不得小于10pt，否则打印不清晰 |
| 联系信息（邮箱/手机） | 10–10.5pt | 略小于正文也可接受 |

**行距**：正文1.2到1.5倍。过密喘不过气，过疏占版面。

---

## 页面与版式

- **纸张**：A4，210×297mm，唯一标准
- **页边距**：上下1.5–2cm，左右1.5–2cm；不要过窄（打印时内容被切掉）
- **分栏**：优先单栏或不等宽双栏（左窄右宽）；等宽双栏信息密度高但阅读跳跃，慎用
- **颜色**：黑白灰最安全；可加一个深蓝色（如#1F3864）做section标题色；禁止高饱和彩色
- **强调方式**：只用加粗一种，不混搭斜体、下划线、高亮

## 照片政策

### 必须放照片
- 国企
- 央企
- 政府机关
- 选调生
- 事业单位
- 国有银行

### 通常不放照片
- 互联网公司（字节、腾讯、阿里、美团等）
- 创业公司
- 外企（大多数）
- 咨询公司

### 不确定时的默认规则
- JD 来自民营互联网：**不放**
- JD 来自国有体制：**放**
- JD 来源不明：**不放**（放错没事，不放错也没大问题）

### 照片规格（需要放时）
- 职业证件照，正装，白色或蓝色背景
- 尺寸：1寸（2.5×3.5cm）或小2寸（3.5×4.5cm）
- 位置：简历右上角

## 模块排序

### 标准排序（通用校招）

1. 个人信息（姓名、联系方式、学校、专业、GPA、排名）
2. 教育背景
3. 项目经历（或实习经历，取最核心的）
4. 校园经历 / 学生工作
5. 技能特长
6. 自我评价

### 排序调整规则

| 场景 | 调整 |
|-----|-----|
| 有实习经验 | 实习经历提到项目经历之前 |
| 学术/科研岗 | 教育背景扩写，科研经历紧随其后，GPA前置 |
| 技术岗 | 技能特长提到校园经历之前 |
| 国企/选调 | 学生工作/党员经历提前，政治面貌在个人信息里前置 |

---

## 政治面貌

- **国企、选调、公务员、事业单位**：必须写，位置在个人信息栏
- **互联网、外企、创业公司**：可省略，写与不写都不影响
- 写法：中共党员 / 共青团员 / 群众
- 党员优先写，这是国企/选调的加分项

---

## 一页原则

- **本科生**：严格一页，无例外
- **硕士生**：默认一页；科研成果极其丰富（2篇以上发表）可酌情两页
- **博士生**：可两页，学术成果较多时正常
- **超页处理顺序**：
  1. 先删弱相关经历（不是压缩字号）
  2. 压缩自我评价（本来就不应该很长）
  3. 每个项目只保留最强2条bullet
  4. 最后才考虑微调字号/行距

---

## 个人信息栏必要字段

必须出现：
- 姓名
- 手机号
- 邮箱

强烈建议出现（HR 高频核查）：
- 学校
- 专业
- GPA / 绩点（有优势时写）
- 毕业年份 / 在读年级

可选（视岗位）：
- 籍贯（选调生必须写）
- 政治面貌（国企必须写）
- GitHub / 作品集链接（技术岗加分）
- 微信号（一般不需要，手机号已够）

**禁止堆砌**：不要把所有信息都放，重点是1秒内可读。

---

## 教育背景写法

- **时间倒序**：最近/最高学历在最前
- 每个学历条目格式：学校 | 专业 | 学历 | 时间
- GPA：写法统一，如"GPA 3.8/4.0"或"3.8/4.0"，不要写"绩点3.8分"
- 排名：写"前X%"比写"第X名/共XXX人"更通用
- 相关课程：与岗位强相关时才列，最多5门，不需要列所有课
- 荣誉/奖学金：单独一行或在教育背景下以bullet列出

---

## 文件命名与导出

- 文件格式：PDF（防排版错位）
- 文件命名推荐：`应聘岗位-姓名-学校专业.pdf`
  - 例：`产品实习-张三-北大信管.pdf`
- 不同岗位保留不同版本，不要维护一个"万能简历"

## 实战补充

### 制作工具建议

- **推荐用 PPT 自行设计**：PowerPoint 排版自由度高，能精确控制每个元素位置，比直接套 Word 模板更灵活
- **不推荐直接套现成网络模板**：现成模板样式与内容强绑定，改起来费力，且雷同率高

### 分点格式规范

- **必须使用项目符号（bullet point 符号）**，不能靠手动换行+缩进模拟"分点"效果；视觉一致性差，且导出 PDF 后对齐往往错乱
- **超过1行的文段缩进2字符**：bullet 第一行顶格，续行缩进2字符，与 bullet 符号对齐

### 颜色使用约束

- **颜色种类 ≤ 2种**：主文本黑色，其余颜色（如色块、标题线）限1种，且应选不刺眼的同色系
- 具体示例：深蓝色系（如 #1F3864）或深灰色系，禁止红色/橙色/绿色等高饱和彩色

### 推荐版式（非等宽双栏）

非等宽双栏简历（左窄右宽）布局建议：

```
┌──────────────┬──────────────────────────────────────────┐
│ 左侧窄栏      │ 右侧宽栏                                  │
│ · 技能特长    │ · 教育背景                                 │
│ · 自我评价    │ · 项目/实习经历                            │
│              │ · 校园经历/学生工作                         │
└──────────────┴──────────────────────────────────────────┘
```

左侧放"支撑性"内容，右侧放"主力"内容，HR 视线从右侧入，左侧作补充。

### 文件命名格式

推荐格式：`【岗位】姓名-学校专业（联系方式）`

- 例：`【产品经理实习】张三-北大信管（138xxxxxxxx）`
- 岗位放括号前置，HR 投递多份简历时一眼区分
- 联系方式放最后，方便快速找到候选人

### 个人信息字段补充规则

以下规则补充现有字段清单：

| 字段 | 建议 |
|-----|------|
| 意向/目标岗位 | **必须写且放显眼位置**；这是人岗匹配的核心信号，没有它 HR 不知道你投的是什么 |
| 邮箱 | 推荐学校邮箱（.edu.cn）；避免 QQ 邮箱；**填了就必须看，不要写了不回** |
| 联系方式 | 留电话，不留微信；手机号已足够 |
| 民族/籍贯/家庭住址 | 几乎无效信息，默认省略；选调生需写籍贯 |
| 性别 | 有照片时可省略（肉眼可辨）；无照片时保留 |
| 政治面貌 | 投高校/国企/事业单位时加；其他岗位可选 |

### 教育背景补充规则

- **只写到本科**：不写高中及以下学历
- **同类荣誉合并**：如"一等奖学金（2021、2022）"，不要逐年单列成两行浪费空间
- **课程要么不写，要写就带成绩**：只写课程名不带成绩没有说服力；如无优秀成绩则省略课程列表
