# 简历写作总规则

## 事实红线（不可逾越）

以下字段必须原样输出，不得修改：

- **学校名称**：不得替换为更强学校，不得简称造歧义
- **学历层次**：本科/硕士/博士，不得升级
- **专业名称**：不得改写成更热门的相关专业
- **学号**：原样（不应出现在简历里，出现时原样保留）
- **GPA / 绩点 / 成绩**：不得上调；排名写"前X%"时不得缩小范围
- **排名 / 作者顺序**：第一作者、第二作者、并列不可互换
- **奖项等级**：国家级/省级/校级/院级不可升级；获奖名次不可提升
- **角色身份**：组长/成员/助理研究员不可随意升格为"负责人""主导"
- **时间区间**：入职/离职/参与时间不可调整
- **项目是否参与**：没做过的项目不可写；参与程度必须可在面试中说清楚
- **数字**：精确数字（样本量、销售额、排名数字）不可篡改

**判断标准**：面试时追问这条，你能否说清楚证据？不能→不写。

---

## 允许的包装

在不违背事实的前提下，以下优化合法：

| 优化类型 | 说明 | 示例 |
|--------|------|------|
| 动词强化 | 把你实际主导的事用更强动词表达 | "参与整理" → "独立完成数据清洗与整合" |
| 数字概括 | 用区间/数量级替代精确小数字 | "55份" → "近60份"；"3%~5%" → "良率提升近一倍" |
| 排序调整 | 把与岗位最相关的经历排前 | 技术岗把项目经历提到校园经历前面 |
| 扩写/压缩 | 关键项目可扩写到4条bullet，弱相关的压缩到1条 | — |
| 结果前置 | 把有数字的结果挪到bullet末尾或开头 | 视上下文决定 |
| 角色合理命名 | 你确实知道项目全貌且承担关键部分时 | "打杂成员" → "核心成员" |

**硬约束**：数字概括不能离谱。55份说"近100份"是造假；说"近60份"可接受。

## 人岗匹配三层推理

agent 在生成前必须完成这三层分析，结论影响排序与侧重：

### 第一层：硬条件必匹配
JD 明确要求的学历/语言/证书/技术栈，候选人没有→如实反映，不要在简历里假装有。

### 第二层：软偏好加分项
JD 里"有XX经验优先"或"了解XX更好"的内容，用户有对应经历→主动提前排序，给更多空间。

### 第三层：JD 暗示风格
企业类型暗示了期望的简历语气（见下节），agent 要据此调整表达方式，不仅是内容。

## 不同场景的语气

### 国企 / 选调 / 事业单位
- 语气：稳重、严谨、有全局观
- 政治面貌：必须写，中共党员优先靠前
- 重点：学生干部职务、党员经历、集体荣誉、班级排名
- 动词偏好：组织、协调、推动、完成、参与
- 禁忌：过于个人化的表达、创业语气

### 互联网 / 科技公司
- 语气：简洁、量化、结果导向
- 政治面貌：可省略
- 重点：项目结果数字、技术栈、业务impact、学习速度
- 动词偏好：主导、构建、优化、落地、迭代
- 禁忌：流水账、形容词堆砌

### 外企
- 语气：国际化、有跨文化视角
- 通常中英文双语或纯英文简历
- 重点：英语能力（雅思/托福分数）、海外经历、跨团队协作
- 禁忌：政治面貌（外企通常不要求）

### 学术 / 科研岗位
- 语气：严谨、论文/项目数量、导师背书
- 重点：GPA前置、科研项目、发表/在投论文、竞赛
- 动词偏好：研究、分析、设计实验、验证假设
- 禁忌：校园活动占太多篇幅

## 常见失败模式

agent 生成bullet前必须对照检查，避免以下问题：

### 流水账
**症状**：逐步描述做了什么，没有结果，没有数字
```
✗ 参加了学校数学建模竞赛，进行了数据收集、模型构建和报告撰写。
✓ 主导数学建模竞赛数据分析模块，搭建回归预测模型，最终获省级二等奖。
```

### 关键词堆砌
**症状**：把JD里的词原封不动塞进去，没有支撑
```
✗ 具备数据分析、项目管理、跨部门协作、沟通能力和用户思维。
✓ 每条技能只保留有具体支撑的，无支撑的删掉。
```

### 自夸形容词
**症状**："优秀的""强大的""出色的"没有事实承载
```
✗ 具有较强的学习能力，能快速掌握新知识。
✓ 三周内自学 Pandas 完成数据清洗，交付1500行分析报告。
```

### 缺量化
**症状**：能加数字却没加，全是定性描述
```
✗ 显著提升了用户满意度。
✓ 满意度调研NPS从62提升至78（+16分）。
```

### 排版过花
**症状**：多种颜色、多种字体、多栏混合、图文嵌套
- agent 不控制排版，但在 bullet 内容里不要写任何排版指令
- 如果模板有多栏，确保内容精简可容纳

### 简历超页
**症状**：本科生两页或更多
- 本科/硕士默认一页
- 超出时优先删弱相关经历，不是压缩字号

## 实战补充

### 可阅读性操作规则

以下规则决定简历是否"好读"，agent 生成内容时应主动遵守：

- **分点是逻辑重构，不是回车换行**：把一段流水账用项目符号拆开，不等于分点。每个bullet必须是独立完整的陈述，而非同一句话被硬切为两行
- **只用加粗一种强调方式**：不混搭斜体、下划线、高亮。过多加粗等于没有加粗——全局加粗的简历视觉噪声极高，等于放弃强调
- **拒绝口语**：不写"我觉得""然后""就是"，不写半句话就收尾

### 可感知性操作规则

以下规则决定 HR 能否快速抓住重点：

- **叙述数字化**：能给数字就给数字；无精确数字时用区间或数量级（见量化优先原则）
- **标题规整化**：每段经历的标题先写概括词再写细节，让人一眼看出是什么（如"产品经理实习 | XX科技 | 2023.06–2023.09"，不要直接写公司名）
- **词句专业化（不硬凹）**：使用行业真实术语，但只用自己真正理解的词；生搬硬凑高大上词汇比不写更糟

### 内容充实度标准

满足以下条件才算"内容充实"：

1. 至少3段经历，每段不少于3行实质内容
2. 至少1段经历与目标岗位的专业技能有直接关联
3. 自我评价不是用来凑版面的——如果没有真正值得写的标签，宁可不写

同时注意另一端：**内容也不要过多**。简历不是档案，不是把所有经历都列上就好。堆砌经历反而让 HR 失去焦点。

### 高频anti-pattern补充

以下错误类型在真实简历中极为常见，agent 生成前必须主动检查：

| 错误类型 | 特征 | 处置 |
|--------|------|------|
| 扬短避长 | 主动写犯错经历、写"熬夜赶进度"等负面细节 | 只写正面结果；过程的辛苦不要写，除非结果极优 |
| 不说人话 | 堆砌"赋能""生态""闭环""顶层设计"等空洞词 | 换成具体行为描述 |
| 岗位说明书化 | 整段描述职责是什么，没有个人行动和结果 | 删去职责通述，只保留"我做了什么→产生了什么" |
| 离谱夸大 | 明显超出学生能力边界的头衔和表述（如"主导公司战略转型"） | 按实际参与深度选动词；宁可低调也不要被面试拆穿 |

### 经历不足时的合法包装方法

用户经历较少时，以下三种方法合法且常见：

1. **打杂经历包装**：如果做过看似简单的辅助工作，找出其中自己独立决策或完成的最小单元，围绕这个单元展开
2. **课程作业作为经历**：课程项目可以写进简历，但必须写清楚做了什么、用了什么方法、产出是什么，不能只写"完成了XX课程"
3. **自学过程作为经历**：如果自学了某个技能并有产出（代码、报告、作品），可以作为单独经历条目，但必须有具体产出作为支撑，不能只写"学习了XX"
