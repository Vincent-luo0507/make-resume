# Bullet 写法规范

## 四段式结构

每条bullet最好覆盖以下四个要素：

```
[动词] + [对象] + [方法/背景] + [结果]
```

agent 生成时必须检查：有没有缺失其中一项。缺结果是最常见问题。

### 示范句

**示范1（项目技术类）**
```
✓ 独立完成用户行为数据清洗与建模，基于随机森林算法识别流失风险用户，
  最终模型准确率达84%，输出给运营团队用于精准触达。
```

**示范2（校园经历/组织管理类）**
```
✓ 主导策划院级迎新晚会全流程，协调15名志愿者分组执行，活动当日参与
  学生超600人，收到书面表扬1份。
```

**示范3（实习/工作类）**
```
✓ 负责竞品分析模块，系统梳理5家竞品的定价与功能矩阵，撰写分析报告
  被产品总监采纳，直接推动2个功能优先级调整。
```

**示范4（研究/学术类）**
```
✓ 参与导师国家自科基金项目，负责文献综述与实验数据录入，整理有效样本
  共300+条，辅助完成1篇在投论文的数据支撑部分。
```

## 动词强度分级

选动词的唯一原则：**只有用户实际承担的职责等级才允许用对应强度的动词**。

用户自己说的内容 → agent 判断实际参与深度 → 选对应级别动词。

### 主导级（仅用于确实独立负责或牵头的事项）
- 主导 / 牵头 / 独立完成 / 负责 / 创立 / 搭建 / 设计

**何时用**：用户是唯一负责人，或在团队里是明确负责人，而非泛泛"参与"。

### 推动级（参与核心，推动了某事发生）
- 推动 / 构建 / 优化 / 落地 / 完成 / 建立 / 梳理 / 输出

**何时用**：用户参与了核心工作，对结果有实质贡献，但不是唯一执行者。

### 参与级（配合他人完成任务）
- 参与 / 协助 / 配合 / 支持 / 跟进 / 整理 / 收集

**何时用**：用户是团队成员，做了具体分工的部分工作，非主导。

### 学习级（以学习为主要目的）
- 学习 / 了解 / 熟悉 / 接触 / 初步掌握

**何时用**：用户是通过课程、自学或旁观获得的能力，无实战产出。技能栏可用，项目经历里慎用。

## 量化优先原则

按以下优先级依次尝试：

1. **有精确数字就用**：样本量、参与人数、金额、准确率、提升点数
   ```
   训练集3000条，测试准确率87.3%
   ```

2. **没精确数字用比例/区间**：
   ```
   效率提升约30%；参与人数超500人；完成度达95%
   ```

3. **没比例用数量级描述**：
   ```
   百人规模的活动；数十份问卷；多轮迭代
   ```

4. **实在无法量化，至少给出范围或对比**：
   ```
   ✗ 显著提升了活动质量
   ✓ 活动满意度高于同期其他社团活动（年度评优中排名第一）
   ```

**禁止量化的情况**：数字明显夸大了事实（见事实红线）。55份问卷不能说"近百份"。

## 反面例句与改写

### 反面1：空洞的"提升效率"

```
✗ 通过优化数据处理流程，显著提升了团队工作效率。

问题：做了什么优化？效率提升多少？谁的效率？没有一个可核实的信息。

✓ 将Excel手工汇总改写为Python脚本自动化处理，原本需1小时的每日报表
  缩短至3分钟，覆盖团队4名成员的日常工作。
```

### 反面2：空洞的"优化流程"

```
✗ 参与优化了部门内部沟通流程，增强了团队协作效果。

问题：参与了什么？优化了什么具体环节？协作效果怎么量化？

✓ 协助建立跨组周会纪要模板，统一输出格式，减少重复确认来回，
  部门周会时长从90分钟压缩至50分钟。
```

### 反面3：空洞的"积累经验"

```
✗ 通过参加比赛，积累了宝贵的团队合作和项目管理经验。

问题：毫无信息量，所有人都能写这句话。

✓ 担任四人小组队长，统筹任务分工与时间节点，带领团队在21天内完成
  完整可运行原型，获校级创新创业大赛三等奖。
```

## 自我评价的bullet规则

自我评价不是形容词堆砌区，必须用"核心标签 + 具体支撑"结构：

```
✗ 本人性格开朗，具有良好的沟通能力，工作认真负责，能承受较大压力。

✓ 具备数据分析与快速建模能力（Python/SQL），有独立交付端到端分析报告经验；
  对产品方向有持续关注，曾系统分析3款竞品并提交可落地改进建议。
```

每条自我评价bullet最多写2句。超过2句就是没有重点。

## 实战补充

### 经历条目万能公式

以下公式适用于项目/实习/校园经历的开头段落：

```
在[背景]情况下，作为[个人定位]，[参与程度副词]参与/负责[项目名称]，
运用[技能]，完成[工作内容/指标]，实现[个人贡献与成效]
```

**各成分说明**：

| 成分 | 说明 | 注意 |
|-----|------|------|
| 背景 | 项目的来源或难度背景（可选），有则写，能强化难度价值 | 不要写废话背景，没有就跳过 |
| 个人定位 | 负责人 / 主要成员 / 核心成员 | **不写"成员"或"普通成员"**；身份是自己给的，选最高真实等级 |
| 参与程度副词 | 全面 / 全程 / 深度 / 独立 / 长期 | 必须与实际相符；"独立"要求真正独立完成 |
| 技能 | 实际使用的工具/方法 | 只写真用过的 |
| 工作内容/指标 | 做了什么、完成程度 | 能量化的量化 |
| 个人贡献与成效 | 产出了什么价值 | 这是最关键的，缺了就是流水账 |

### 每段经历的长度约束

- 每段经历正文：**2–3整行**（太短无信息量，太长 HR 不读）
- 分点数量：**≤3个bullet点**，每点**≤2行**
- 违反约束时：优先删弱 bullet，不是压缩字号

### 软素质表达模板

软素质（学习能力、团队精神等）不能直接声称，必须通过具体句式间接表达。以下是已验证有效的写法：

**学习能力**

- 快速上手型：`入职/加入X天内独立完成[具体任务]，[量化成果]`
- 持续主动型：`在[项目/工作中]主动学习[技能]，并将其应用于[具体场景]，[产出]`

**创新能力**

- 标准句式：`在[场景]中首次提出[方案]，经[验证方式]证明可行，[成效]`
- 核心要素：敢为人先 + 卓有成效（两者缺一才有说服力）

**逻辑能力**

- 推理分析型：`面对[问题]，拆解为[N个子问题]，逐一分析后得出[结论]，[被采纳/产生影响]`
- 归纳总结型：`梳理[X份/X轮]材料/数据，归纳出[N个规律]，形成[报告/方案]`

**团队精神**

- 领导者版：`带领[N人]团队，明确分工，[N天/周]内完成[目标]，[结果]`
- 协作者版：`在[N人]团队中负责[具体模块]，与[角色]保持[频率]同步，确保[交付质量]`

**脚踏实地**

- 标准句式：`从[起点]出发，通过[具体路径]，历经[时间/轮次]，达成[目标]`
- 核心要素：路径明确 + 持续攻关（体现的是耐力，不是态度宣言）
